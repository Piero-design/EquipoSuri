Origen y Problema

Creado en 2017 por el desarrollador francés Regis Freyd, Monica nació de un problema personal: olvidaba detalles importantes sobre las personas que le importaban.

"Me estaba convirtiendo en un mal amigo. Olvidaba los nombres de los hijos de mis amigos y las conversaciones importantes."

A diferencia de los CRM empresariales orientados a ventas, Monica es un CRM Personal: 100% privado, ético y sin algoritmos. Es el "Anti-Facebook".

🏗️

Arquitectura MVC

El sistema sigue un patrón Model-View-Controller estricto con Laravel. Nuestro foco de testing ataca directamente la capa de Services.

🖥️ Blade + Vue.js → Presentación
⚙️ Controllers → Services / Actions Foco Testing
🗄️ Eloquent ORM → MySQL → Persistencia
🛠️

¿Para qué sirve Monica?

Monica gestiona las relaciones personales del usuario, no las comerciales. Principales funcionalidades:

  • 📇 Contactos: Nombres, relaciones, fotos, etiquetas y datos personales
  • 🔔 Recordatorios: Cumpleaños, fechas importantes y eventos recurrentes
  • 📝 Actividades: Registro de interacciones, llamadas y reuniones
  • 📓 Journal: Diario personal de experiencias y anotaciones
  • 📊 Vault: Archivos, métricas y reportes del usuario
  • ⚙️ Configuración: Preferencias, plantillas y gestión de usuarios
🧪

Herramientas de Testing

Nuestro proceso de calidad se apoya en herramientas especializadas integradas en el ecosistema.

  • ✓ PHPUnit: Framework principal para pruebas unitarias de backend.
  • ✓ Mockery: Simulación (mocking) de servicios y dependencias externas.
  • ✓ SQLite In-Memory: Base de datos en memoria RAM para pruebas rápidas.
  • ✓ GitHub Actions: Pipeline automático de Integración Continua (CI/CD).
